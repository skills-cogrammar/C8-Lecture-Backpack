7. Loops and Iterations (20 mins)

Guide:

  • Create a file loops.py
  • Explain and demonstrate for loops:
# Sum of numbers in a fixed list
numbers = [1, 2, 3, 4, 5]
total = 0
for num in numbers:
    total += num
print(f"Sum using for loop: {total}")
  • Explain and demonstrate while loops:
# Sum of numbers in a list with variable size
numbers = []
while True:
    num = input("Enter a number (or 'done' to finish): ")
    if num == 'done':
        break
    numbers.append(int(num))

total = 0
i = 0
while i < len(numbers):
    total += numbers[i]
    i += 1
print(f"Sum using while loop: {total}")

8. Full Coding Exercise (25 mins)

Guide:

  • Create main.py with the following content:
# main.py
# Data Types & Input
name = input("Enter your name: ")  # String
age = int(input("Enter your age: "))  # Integer
is_student = input("Are you a student? (yes/no): ").lower() == 'yes'  # Boolean

# Conditional Statements
if age < 18:
    print(f"Hello {name}, you are a minor.")
elif 18 <= age < 65:
    print(f"Hello {name}, you are an adult.")
else:
    print(f"Hello {name}, you are a senior citizen.")

# While Loop: Countdown
count = 3
while count > 0:
    print(f"Starting in {count}...")
    count -= 1
print("Go!")

# For Loop: Multiplication Table
print("\nMultiplication Table (1-5):")
for i in range(1, 6):
    for j in range(1, 6):
        print(f"{i} x {j} = {i * j}")
    print("")  # Blank line for separation
  • Explain each section of the code
  • Run the script and demonstrate the output

9. Push Code to GitHub (10 mins)

Guide:

  • Stage changes:
git add loops.py main.py
  • Commit changes:
git commit -m "Add loops example and main exercise"
  • Push to GitHub:
git push -u origin main
  • View changes on GitHub repository

10. GitHub .dev Environment (10 mins)

Guide:

  • Explain the concept of GitHub's .dev environment
  • Demonstrate how to access it:
    • Navigate to your GitHub repository
    • Press the '.' key or change .com to .dev in the URL
  • Highlight key features:
    • Web-based VS Code environment
    • No need for local VS Code installation
    • Access to full repository
    • Ability to edit, commit, and push changes directly from the browser
  • Show how to:
    • Open files
    • Make edits
    • Commit changes
    • Push to the repository
  • Discuss benefits:
    • Quick edits without local setup
    • Consistent development environment
    • Great for collaboration and code reviews
    • Useful for quick demonstrations or when away from your main development machine
